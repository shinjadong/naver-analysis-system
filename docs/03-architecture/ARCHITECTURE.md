# 시스템 아키텍처

> **마지막 업데이트**: 2025-12-15
> **버전**: 0.4.0 (Persona Management + Portal 통합 계획 반영)

---

## 1. 개요

Naver AI Evolution은 Windows-WSL2 하이브리드 환경에서 동작하는
다중 안드로이드 디바이스 AI 에이전트 통합 관리 시스템입니다.

### 1.1 핵심 목표

1. **탐지 회피**: 네이버의 다층적 추적 시스템(74개 도메인, 40개 쿠키) 대응
2. **자가 진화**: 유전 알고리즘 기반 전략 최적화
3. **인간 행동 모방**: 베지어 커브 터치, 가변 스크롤, 자연스러운 타이핑
4. **AI 기반 복구**: DeepSeek API 활용 에러 분석 및 자동 복구

#### 3.3.2 신규 구현 상세

**NaverChromeUse** (`naver_chrome_use/`)
- `provider.py`: NaverChromeUseProvider - 브라우저 선택 및 카드 제공
- `url_builder.py`: NaverUrlBuilder - 네이버 URL 생성 헬퍼
- `cards/`: Chrome/Samsung Internet 브라우저 카드

**Session Manager** (`session_manager/`)
- `device_session_manager.py`: DeviceSessionManager
  - IP 회전 (비행기 모드/모바일 데이터 토글)
  - Chrome 데이터 삭제 (NNB 쿠키 초기화)
  - 30분 쿨다운 관리
- `engagement_simulator.py`: EngagementSimulator
  - 자연스러운 블로그 방문/읽기 패턴
  - 검색 → 결과 클릭 → 콘텐츠 읽기 시뮬레이션

**Device Tools** (`device_tools/`)
- `behavior_injector.py`: BehaviorInjector - 베지어 커브, 가변 스크롤, 인간적인 타이핑
- `adb_enhanced.py`: EnhancedAdbTools - 탐지 회피 탭/스크롤/입력 (모든 입력의 단일 진입점, 휴먼라이크 강제)

## 4. 데이터 흐름

### 4.1 기본 데이터 흐름

```
1. 사용자 → Dashboard → API 요청
2. API → Task Scheduler → 작업 큐
3. Task Scheduler → Device Manager → 디바이스 선택
4. Device Manager → ADB → 명령 실행
5. 결과 → Error Handler → 에러 분석/복구
6. 결과 → Evolution Engine → 학습/최적화
7. 결과 → Dashboard → 사용자
```

### 4.2 네이버 추적 대응 흐름

```
1. 작업 요청 수신
2. Cookie Manager → 쿠키 프로파일 로드/생성
3. Session Simulator → SRT 쿠키 갱신 스케줄링
4. Behavior Simulator → 자연스러운 액션 시퀀스 생성
   - 베지어 커브 터치 궤적
   - 가변 속도 스크롤
   - 오타 포함 타이핑
5. Device Manager → ADB 명령 실행
6. Detection Evasion → 추적 응답 분석
   - 위협 수준 평가
   - 실시간 행동 조정
7. Evolution Engine → 피트니스 평가 (12개 지표)
8. 다음 세대 전략 생성
```

### 7.2 쿠키 갱신 전략

| 쿠키 | 갱신 주기 | 전략 |
|------|----------|------|
| NNB | 30일 | 디바이스 핑거프린트와 일관성 유지 |
| NID | 로그인 시 | 계정 정보와 일치 |
| SRT5 | 5분 | 활동 기반 갱신 시뮬레이션 |
| SRT30 | 30분 | 세션 지속 시뮬레이션 |
| BUC | 실험 노출 시 | 행동 변화 연계 |
