# 네이버 브라우저 지문 캡처 실험 프로토콜

## 목적
네이버가 실제로 수집하는 브라우저/디바이스 지문 데이터 확인
→ IP + 쿠키 변경만으로 "다른 사용자"로 인식되는지 검증

---

## 실험 환경

### 캡처 장비
- **디바이스**: 루팅된 갤럭시탭
- **캡처 도구**: (예: PCAPdroid, tPacketCapture, mitmproxy 등)
- **캡처 범위**: HTTPS 트래픽 전체 (SSL 복호화 필요)

### 테스트 디바이스
- **디바이스**: Galaxy Z Fold5 (SM-F946N)
- **브라우저**: Chrome
- **네트워크**: LTE/5G (WiFi 끔)

---

## 실험 시나리오

### [실험 A] 기본 세션 - 첫 번째 "사용자"

```
1. 캡처 시작
2. Chrome으로 네이버 홈 접속 (m.naver.com)
3. "맛집 추천" 블로그 검색
4. 첫 번째 결과 클릭
5. 5회 스크롤 (각 1-2초 대기)
6. 뒤로가기
7. 캡처 종료 → session_a.pcap 저장
```

### [실험 B] 세션 리셋 후 - "새로운 사용자" 시뮬레이션

```
1. 비행기 모드 ON (5초 대기)
2. 비행기 모드 OFF (새 IP 할당 대기 10초)
3. Chrome 데이터 전체 삭제
   - adb shell pm clear com.android.chrome
4. 캡처 시작
5. Chrome으로 네이버 홈 접속 (m.naver.com)
6. "서울 카페" 블로그 검색 (다른 검색어)
7. 두 번째 결과 클릭 (다른 위치)
8. 3회 스크롤 (다른 패턴)
9. 뒤로가기
10. 캡처 종료 → session_b.pcap 저장
```

### [실험 C] 쿠키만 삭제 (IP 유지)

```
1. Chrome 데이터 전체 삭제 (IP는 그대로)
2. 캡처 시작
3. 동일한 시나리오 반복
4. 캡처 종료 → session_c.pcap 저장
```

## DeepSeek 분석 프롬프트

```
당신은 네이버의 사용자 추적 시스템을 분석하는 보안 연구원입니다.

아래 네트워크 캡처 데이터를 분석해주세요:

[세션 A 데이터]
{session_a 캡처 내용}

[세션 B 데이터 - IP 변경 + 쿠키 삭제 후]
{session_b 캡처 내용}

[세션 C 데이터 - 쿠키만 삭제]
{session_c 캡처 내용}

## 분석 요청사항

1. **쿠키 비교**
   - NNB 쿠키 값이 세션마다 다른가?
   - 새 NNB가 생성되는 조건은?

2. **브라우저 지문 추출**
   - TIVAN/VETA 요청에서 전송되는 지문 데이터 목록
   - 각 지문의 해시값 또는 원본값
   - 세션 A, B, C에서 동일한 지문이 있는지

3. **디바이스 식별자**
   - 광고 ID(GAID) 전송 여부
   - Android ID 전송 여부
   - 기타 하드웨어 식별자

4. **핵심 질문 답변**
   - Q1: IP + 쿠키만 바꾸면 네이버는 "다른 사용자"로 인식하는가?
   - Q2: 브라우저 지문으로 "같은 디바이스"임을 알 수 있는가?
   - Q3: 루팅 없이 변경 불가능한 식별자가 전송되는가?

5. **인게이지먼트 관점**
   - 블로그 조회수가 별도로 카운트되는 조건
   - 동일 사용자의 재방문 vs 신규 방문 구분 기준

JSON 형식과 표로 정리해주세요.
```
